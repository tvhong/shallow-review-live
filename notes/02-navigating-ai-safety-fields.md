# Helping People Navigate AI Safety Fields

**Hackathon focus** — this is what we're building now.

## Project shape

Two possible approaches:

1. **Claude Code integration** — hook up all the data sources and links into a markdown file, add it to Claude Code. Simple, fast to build, validates the idea.
2. **Slack/Discord bot** — integrate a chatbot into Slack or Discord where people already hang out. Low friction, meets users where they are.
3. **Website UI** — a dedicated web interface to browse/explore the data. More polished, but more work.

**Decision:** Start with the Claude Code integration. It's the simplest and validates the core question — can people actually use this data to navigate AI safety fields? Slack/Discord bot and website UI can come later.

## Milestone 1: Theory of change across AI safety fields

- Grade/surface **theory of change** across different AI safety fields
- Make it easy for people to explore in a navigable format
- Could look like adding new metadata fields to the docs
- Bottleneck: getting high-quality data — but probably tractable

## Milestone 2: Skill roadmaps

- Once someone identifies where they fit via theory of change...
- Help them figure out **how to get there**
- Recommend a roadmap for developing the required skills, e.g.:
  - Self-learning / engineering skill development
  - Finding mentorship
  - Relevant programs to apply to
- Naturally follows from Milestone 1
