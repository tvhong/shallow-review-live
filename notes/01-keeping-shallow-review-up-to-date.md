# Keeping the Shallow Review Up-to-Date

Lower priority for now — data that's 3–6 months stale is probably fine.

## Pipeline Steps

1. **Scrape the data** — collect new/updated information from sources
2. **Review the data** — someone reviews the scraped data for quality
3. **Update the document** — incorporate changes as appropriate

## Open Challenges

### 1. Time cutoff for a yearly review format

- The original structure is a yearly review
- New data may be useful, but how do we decide the cutoff point?
- Need a clear policy on what counts as "this year's" data vs. next year's

### 2. Reviewing updated content (the real bottleneck)

- We can use LLMs to **propose** updates to the document
- A human still needs to **review** the LLM-proposed changes
- Key question: **how do we make human review easier?**
  - This is likely the hardest problem to solve going forward

### 3. Taxonomy of the field

- The taxonomy of AI safety fields may itself need updating — need to be careful about how we handle this
- Changes to taxonomy have downstream effects on the rest of the review
